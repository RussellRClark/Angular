Deployments here

  • FireBase

    Firebase is the easiest hosting to get running and it has a free tier. The project redeploys from the dist folder on each production build . Tests are run manually in development.

  • GitHub Pages

    GitHub and GitHub Pages require more installation & deployment steps than FireBase because the code is held in a repository. From development a build step and a git commit step are required. Tests are run manually in development. GitHub offers a free tier (used here) and Pipelines & Services which are not used here.

  • Netlify

    Like Firebase, Netlify has a free tier, is easy to install & deploy from the command line. Note the use of the _redirects file for SPA applications. Continuous Deployment is also available on Git push by linking to a GITHub repository & specifying a build command. Tests are run manually in development.

Continuous Integration & Deployment

  • GitLab

    The code hosted in this (free) public repo is deployed to a FireBase development site & a FireBase production site. GitLab runs the Unit Tests, Integration Tests & the e2e (end-to-end) UI tests; and then deploys the code. See the ReadMe in the repository for more details.

  • FireBase

    This is the production site deployed from GitLab. The development site is not shown.

  • Heroku

    Heroku offers a free tier with Continuous Deployment & CI & CD via its pipelines. Heroku requires a few code changes as it uses Node.js to run the site. See the ReadMe in the repository for more details.
