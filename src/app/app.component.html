Continuous Deployment

Code is deployed on Push to the repository or manually from the development terminal
Tests are deployed manually from the development terminal

  • Code Repository: GitHub

    The code hosted in this (free) public repo is deployed to GitHub Pages, Heroku, Netlife and FireBase.
    See the ReadMe in the repository for more details.

  • Host: FireBase

    Firebase is the easiest hosting to get running and it has a free tier.
    The project redeploys from the dist folder on each production build .
    Tests are run manually in development.

  • Host: GitHub Pages

    GitHub and GitHub Pages require more installation & deployment steps than FireBase because the code is held in a repository.
    From development a build step and a git commit step are required. Tests are run manually in development.
    GitHub offers a free tier (used here) and Pipelines & Services which are not used here.

  • Host: Netlify

    Like Firebase, Netlify has a free tier, is easy to install & deploy from the command line. Note the use of the _redirects file for SPA applications.
    Continuous Deployment is also available on Git push by linking to a GITHub repository & specifying a build command. Tests are run manually in development.
    I have redirected one of my domains to this site, both HTTP & HTTPS following Netlify's simple, clear instructions.

  • Host: Heroku

    Heroku offers a free tier with Continuous Deployment & CI & CD via its pipelines. Heroku requires a few code changes as it uses Node.js to run the site.
    See the ReadMe in the repository for more details.
